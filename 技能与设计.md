# 二、前端设计技能（AI 参考）

### 设计思考
- **目的**：界面解决什么问题？谁是主要使用者？
- **气质**：选择明确方向（极简/极繁/复古/有机/杂志风等）
- **差异化**：用户离开后最记住的点是什么？

### 美学原则
- **字体**：有个性、有审美价值，禁止 Arial/Inter 等泛滥字体
- **配色**：主色明确 + 少量锐利强调色，使用 CSS 变量
- **动效**：关键时刻的精心编排，胜过零散小动效
- **布局**：打破常规，非对称/重叠/斜向，避免中庸

### 严禁事项
- 过度使用的字体（Inter、Roboto、Arial）
- 白底 + 紫色渐变等陈词滥调配色
- 缺乏上下文与性格的模板化设计

### 核心理念
- 设计必须为当前场景量身定制
- 极繁需复杂代码与丰富效果，极简需高度克制与精准细节
- 优雅是恰到好处，不是少

前端更新策略：
本技能用于迭代与重塑现有前端设计，目标不是延续旧范式，而是建立更优雅、更克制、更一致的新设计体系。

强调：
从“能用”升级为“好用 + 好看 + 可长期演进”
从“组件堆叠”升级为“结构清晰、信息优先”
从“模板感 UI”升级为“有气质、有判断的产品界面”
需要实现真实可运行、可维护、可扩展的前端代码，而非设计稿或概念示意。

设计前提（Design Premise）
这是一次设计演进（Evolution），而非维护（Maintenance）：
可以推翻原有布局
可以重组信息层级
可以重做组件结
可以删减冗余视觉与交互
唯一不能做的事：在平庸范式上微调。
设计思考（Design Thinking）
在编码前，必须先完成以下判断：

目的（Purpose）
当前界面最核心的信息任务是什么？
用户进入页面后的 10 秒内 应该看懂什么？

使用者（Audience）
高频用户还是偶发访问？
扫读为主，还是深度阅读 / 操作？

更新目标（Evolution Goal）
当前设计“显得普通”的原因是什么？
信息噪音过多？
视觉层级混乱？
组件风格不统一？
交互打断思路？

本次设计必须至少解决其中一个根因。
整体设计取向（Overall Direction）
核心气质
克制（Restrained）
信息优先（Information-first）

一致（Consistent）
轻但不空（Light, not empty）
避免浮夸、避免炫技、避免“设计感过度可见”。
页面结构规范（Layout System）
默认采用 清晰、可预期的结构：

顶部：header + subtitle + top-nav
主体：简洁清晰易读
底部：统一 footer
页面结构应让用户无需学习即可理解。
模块设计规范（Module Rules）

列表类模块：
mini-section + mini-list
→ 强调可扫读性与节奏

行为类模块：
顶部 toolbar，关键行为使用 primary

提示与说明：
使用 section-hint，弱化呈现，不喧宾夺主
组件设计以可复用、一致性优先，而非追求“每块都不一样”。

交互策略（Interaction Strategy）
默认 不使用弹窗，除非涉及不可逆确认
加载态 / 空状态 简洁、低存在感
偏好设置、本地缓存 优先使用本地存储
交互应不中断用户阅读或思考

前端美学原则（Aesthetic Principles）
字体（Typography）
字体必须服务信息清晰度
选择有气质但不过度抢戏的字体

禁止使用：
Arial
Inter
Roboto
字号、行高、层级要稳定且可预期

配色（Color System）
主色明确，强调色极少
浅对比、柔和阴影、统一圆角
使用 CSS Variables 管理颜色体系

颜色用于区分信息层级，而非装饰
布局与空间（Spacing & Layout）
留白是信息的一部分
模块之间有清晰节奏
避免“为了设计感而打破结构”

动效（Motion）
动效用于帮助理解层级与状态
避免装饰性动画
一个克制、精确的过渡 > 多个炫技动画

严禁事项（NEVER）
延续旧页面的随意结构
为了“看起来高级”而增加复杂度
模板化组件拼接
过度设计、过度强调视觉存在感
白底 + 紫色渐变等 AI 常见套路

实现要求（Implementation）
最终交付必须满足：
可直接上线
结构清晰、逻辑自洽
设计规范可复用
能作为后续页面的设计母板

核心理念（Core Philosophy）

前端设计的成熟标志不是“更炫”，
而是：
界面变得安静，但更有力量。
本技能用于帮助你从“还可以”走向“很优雅”，
并建立一个可以长期演进的前端设计基准。
