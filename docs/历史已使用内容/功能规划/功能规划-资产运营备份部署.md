# 功能规划-资产运营备份部署

## 现状
- 缺少账号资产（收藏/关注/订阅/通知）。
- 冷启动仅依赖静态种子，缺周期补充与运营入口。
- 缺少明确的部署文档与备份流程约束。

## 目标
- 增加账号资产与通知体系。
- 增加管理员运营机制（公告、置顶/精选、手动补充话题）。
- 增加冷启动自动补充策略。
- 增加部署文档与备份脚本，并纳入 SOP。

## 实施拆分

| 步骤 | 内容 | 状态 |
| --- | --- | --- |
| 1 | 数据库 schema 扩展（资产、通知、公告、设置） | ✅ 已完成 |
| 2 | 后端服务与路由（收藏/关注/订阅/通知/运营） | ✅ 已完成 |
| 3 | 前端页面与交互（个人中心扩展、收藏/订阅/关注、通知） | ✅ 已完成 |
| 4 | 冷启动策略（空库种子 + 周期补充 + 管理员触发） | ✅ 已完成 |
| 5 | 备份脚本与 SOP 同步 | ✅ 已完成 |
| 6 | 部署文档（Docker / 传统） | ✅ 已完成 |
